## 二、构建问题及修复记录

### 问题 1: Geist 字体不兼容
- **症状**: `next/font` error: Unknown font `Geist`
- **原因**: shadcn init 添加了 `Geist` 字体，该字体仅 Next.js 15+ 支持，项目使用 Next.js 14
- **修复**: 移除 Geist 导入，仅保留 Inter 字体

### 问题 2: `@import "shadcn/tailwind.css"` 与 Tailwind v3 冲突
- **症状**: `The border-border class does not exist`
- **原因**: shadcn v4 的 CSS 导入方式与 Tailwind CSS 3.x 不兼容
- **修复**: 移除该导入，在 globals.css 中直接定义 CSS 变量 `--color-background`、`--color-border`、`--color-card`

### 问题 3: MagicUI 组件使用 named export
- **症状**: `does not contain a default export`
- **原因**: MagicUI 所有组件均使用 `export function`/`export const`（命名导出）
- **修复**: 全局替换 `import X from` → `import { X } from`（涉及 13 个文件）

### 问题 4: `@theme inline` 语法不支持
- **症状**: Tailwind v3 无法解析 `@theme` 指令
- **原因**: `@theme` 是 Tailwind CSS v4 语法
- **修复**: 将 keyframes 和 animation 类定义迁移到 `@layer base` 和 `tailwind.config.ts`

### 问题 5: DOM 内置 `Comment` 类型冲突
- **症状**: `Type 'Comment[]' is not assignable to type 'Comment[]'`
- **原因**: TypeScript 将 `Comment` 解析为 DOM API 的 `Comment` 接口而非自定义类型
- **修复**: 重命名为 `BlogComment`

### 问题 6: `formatDate` / `getTagColor` 函数缺失
- **症状**: `is not exported from '@/lib/utils'`
- **原因**: shadcn init 覆盖了 utils.ts，之前存在的工具函数丢失
- **修复**: 在 `utils.ts` 中重新添加 `formatDate` 和 `getTagColor`

## 四、人工操作指引

### 1. 本地启动开发环境

**后端**:
```bash
cd blog-backend
# 确保 JAVA_HOME 已设置
./mvnw spring-boot:run
# 服务启动于 http://localhost:8080
```

**前端**:
```bash
cd blog-frontend
npm run dev
# 访问 http://localhost:3000
```

### 2. Docker 容器化部署

```bash
# 1. 构建后端 JAR
cd blog-backend
./mvnw clean package -DskipTests

# 2. 构建前端（standalone 模式，会自动启用）
cd blog-frontend
npm ci && npm run build

# 3. Docker Compose 启动
cd ..
docker-compose up -d --build
```

### 3. 端口冲突处理

**场景：本地已有 MySQL 占用 3306/3307 端口**
- Docker 的 MySQL 服务默认注释，不启用
- 如需启用，在 `docker-compose.yml` 中映射 `3308:3306`

**场景：本地已有服务占用 8080 端口**
- `docker-compose.yml` 已设主机侧 8081 → 容器 8080：
  ```yaml
  ports:
    - "8081:8080"
  ```
- 如需再改，直接编辑左侧数字

**场景：3000 端口已被其他 Docker 项目占用**
- `docker-compose.yml` 已设主机侧 3002 → 容器 3000：
  ```yaml
  ports:
    - "3002:3000"
  ```
  ```yaml
  environment:
    - BACKEND_URL=http://backend:8080  # 内部端口不变
  ```

**场景：本地已有服务占用 3000 端口**
- 修改 `docker-compose.yml` 中 frontend 的端口映射：
  ```yaml
  ports:
    - "3001:3000"  # 左侧改为 3001
  ```

### 4. 数据库切换：H2 → MySQL

修改 `blog-backend/src/main/resources/application.yml`:

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/starblog?useSSL=false&serverTimezone=Asia/Shanghai
    driver-class-name: com.mysql.cj.jdbc.Driver
    username: root
    password: yourpassword
  jpa:
    hibernate:
      ddl-auto: update  # 首次设为 update 自动建表
```

同时在 `docker-compose.yml` 中取消 MySQL 服务的注释。

### 5. 上传文件目录

后端文件上传路径为 `./upload/`（相对工作目录），已映射为 Docker volume `blog-upload`。
本地开发时，确保该目录存在或由 Spring Boot 自动创建。
静态资源访问路径为 `http://localhost:8080/uploads/{filename}`。
